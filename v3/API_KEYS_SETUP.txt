======================================================================
 Pythos v3 - Setup & Environment (Keys are OPTIONAL)
======================================================================

Pythos runs its AI brain ENTIRELY ON-DEVICE. The language model, tool
calling, and screen vision all run locally through Ollama + Gemma 4 —
nothing about your prompts or screen ever leaves the machine, and NO
cloud LLM API key is required.

This is a PUBLIC repo, so no keys are committed. The only keys below are
for OPTIONAL cloud enhancements (studio-quality voice, optional cloud
Gemma via OpenRouter). Web search uses the built-in free DuckDuckGo HTML
path and does not need an API key. The Python worker inherits the
environment of whatever shell launches the app, so any keys you do use
must be exported in that shell BEFORE you start Pythos.


----------------------------------------------------------------------
 REQUIRED: local model runtime (no API key, just a local install)
----------------------------------------------------------------------

The brain is Gemma 4 served by Ollama on your own machine.

  1. Install Ollama:            https://ollama.com/download
                                (macOS: `brew install ollama`)
  2. Start the server:          `ollama serve`   (runs on 127.0.0.1:11434)
  3. Pull the model(s):         `ollama pull gemma4:12b`
                                `ollama pull gemma4:e2b`   (optional, low-resource mode)

     Or use the helper script:  ./scripts/install-ollama-models.sh
                                .\scripts\install-ollama-models.ps1

  gemma4:12b  (~7.6 GB, 256K context) is the default and best at tool calls.
  gemma4:e2b  (~1-2 GB) is the low-resource fallback — enable "Low resource
              mode" in Settings (or set python.lowResourceMode=true in
              config.json) to use it. It must be pulled first.

That is the ONLY hard requirement for the assistant. Typed chat, tools,
sub-agents, and screen vision all work with just Ollama + Gemma, offline.


----------------------------------------------------------------------
 OPTIONAL: cloud voice (studio-quality STT + TTS when online)
----------------------------------------------------------------------

  GRADIUM_API_KEY   -> Studio-quality streaming voice (get it at
                       https://studio.gradium.ai)

When GRADIUM_API_KEY is set and the network is up, wake word, STT, and
TTS stream through the Gradium cloud API for studio quality. When the key
is missing OR the network drops, Pythos automatically falls back to fully
local voice: Vosk STT + Piper (if installed) or the OS system voice
(macOS `say`, Windows System.Speech, Linux espeak-ng). The Gemma brain
path never changes. Typed prompts always work offline with no key.

Install the offline ASR model once for spoken input without Gradium:
  ./scripts/install-vosk-model.sh        # macOS / Linux
  .\scripts\install-vosk-model.ps1        # Windows PowerShell


----------------------------------------------------------------------
 OPTIONAL: cloud Gemma via OpenRouter
----------------------------------------------------------------------

  OPENROUTER_API_KEY  -> Your personal key from https://openrouter.ai/keys
  OPENROUTER_MODEL    -> default google/gemma-4-31b-it:free

Enable in Settings → General → LLM provider → OpenRouter, then paste your
API key. The key is stored in your local user-settings file (never in git).
You can also set OPENROUTER_API_KEY in .env. When OpenRouter is enabled,
typed chat and tool loops use the cloud model instead of local Ollama.


----------------------------------------------------------------------
 FREE: web search (no API key)
----------------------------------------------------------------------

Web search and deep research use DuckDuckGo HTML parsing built into
localTools.ts. No BRAVE_API_KEY, PYTHOS_WEB_SEARCH_KEY, or MCP search
server is required. Results are summarized for spoken output.


----------------------------------------------------------------------
 OPTIONAL: other integrations
----------------------------------------------------------------------

  PYTHOS_OLLAMA_MODEL    -> override the Gemma model (default gemma4:12b)
  PYTHOS_OLLAMA_URL      -> override the Ollama endpoint (default
                            http://127.0.0.1:11434)
  PYTHOS_CURSOR_WORKSPACE -> project directory for Cursor agent CLI tasks
  PYTHOS_MCP_SANDBOX     -> root directory MCP file tools may read/write
  SPOTIFY_CLIENT_ID      -> Spotify control (a default client ID is in
                            config.json; set your own for production)
  GITHUB_PERSONAL_ACCESS_TOKEN -> only if you enable the github MCP server

Note: GEMINI_API_KEY / GOOGLE_API_KEY are NO LONGER USED by the core
assistant (it is fully local now). They are only relevant if you manually
enable the experimental Pi tool bridge (pi.enabled=true in config.json),
which is off by default.


----------------------------------------------------------------------
 Quick start: create a local .env file (optional)
----------------------------------------------------------------------

The app does NOT auto-load .env; export the vars in your shell, or source
the file before launching. The file is gitignored so it will NOT commit.

# macOS / Linux (bash or zsh):
cd /path/to/RAISE-Hackathon/v3
cp .env.example .env
echo 'GRADIUM_API_KEY=your_key_here' >> .env      # only if you want cloud voice
set -a; source ./.env; set +a
npm run dev

# Windows - PowerShell:
cd path\to\RAISE-Hackathon\v3
Copy-Item .env.example .env -Force
Add-Content .env "GRADIUM_API_KEY=your_key_here"   # only if you want cloud voice


----------------------------------------------------------------------
 Verify your setup
----------------------------------------------------------------------

  Ollama up + model pulled:
    curl http://127.0.0.1:11434/api/tags        # should list gemma4:12b

  Voice key visible (only if using cloud voice):
    macOS / Linux:   echo $GRADIUM_API_KEY
    PowerShell:      echo $env:GRADIUM_API_KEY

  Offline voice (no key needed):
    Install Vosk model (see above), launch app — HUD shows
    "Voice: Local (offline)" or "Voice: Gradium (cloud)".

If `ollama list` shows gemma4:12b, launch the app and start talking (or
typing) — the brain is already running locally.
